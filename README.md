# 二维码/条形码生成与识别工具

## 项目定位

本项目计划基于 Qt 构建桌面端图形界面，并逐步接入 OpenCV 及后续条码识别/编码适配层，形成一个同时支持二维码与条形码生成、图片识别和摄像头实时识别的工具。

当前提交只建立最初架构，重点是把界面层、业务服务层和基础数据类型分开，便于后续补充真实编解码能力。

## 初始目录结构

```text
.
├── CMakeLists.txt
├── include/ptapro/
│   ├── core/              # 编解码通用数据结构与枚举
│   ├── services/          # 生成、识别等业务服务接口
│   └── ui/                # Qt 主窗口声明
└── src/
    ├── core/
    ├── services/
    └── ui/
```

## 模块职责

- `core`：定义码制、编码请求、编码结果、识别结果等跨模块数据结构。
- `services`：封装生成与识别流程。当前提供占位实现，后续可在这里接入 QRCode、ZXing、ZBar 或 OpenCV 适配层。
- `ui`：负责 Qt Widgets 界面、用户输入、文件选择和结果展示，不直接依赖具体第三方编解码实现。

## 构建方式

```bash
cmake -S . -B build
cmake --build build
```

说明：Qt Widgets 是当前最小必需依赖；OpenCV 在初始架构中是可选依赖，缺失时不会影响项目配置和界面骨架编译。

## 后续计划

1. 接入真实二维码生成库，替换当前占位渲染。
2. 接入条形码生成能力，优先支持 Code 128、EAN-13 等常见码制。
3. 实现图片文件识别，并统一返回识别文本、码制和错误信息。
4. 增加摄像头实时识别入口。
5. 补充核心服务单元测试和典型图片样本测试。
